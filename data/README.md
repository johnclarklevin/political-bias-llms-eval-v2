# Data guide

## Primary files

- `responses.csv`: all 450 physical GPT-5.5 responses joined to their GPT-5.6 Sol classifications, prompts, endpoints, model IDs, response IDs, usage records, and timestamps.
- `raw/generations.jsonl`: complete generation records.
- `raw/judgments.jsonl`: complete judge records.
- `raw/manifest.json`: run configuration and exact system messages.
- `summary-statistics.csv`: the five topline rows shown in the chart.
- `local-arm-statistics.csv`: counts, shares, and mean response length for the four local analysis rows.
- `topic-summary.csv`: results by topic and condition.
- `analysis-rows.csv`: physical responses plus the derived No Fringe Questions rows.
- `no-fringe-assessment.csv`: 30-topic support assessment and source URLs.
- `label-verification-sample.csv`: deterministic 20-response manual-audit sample.
- `long-response-label-audit.csv`: deterministic 30-response audit across both long-response arms and their two common labels.
- `washington-post-consistency-runs.csv`: the Post’s supplementary automated five-run GPT comparison values; these are not the main-chart row.
- `judge-validation/results.jsonl` archives the marker-stripped validation attempts: 180 retained network failures followed by 180 successful calls. `judge-validation/summary.json` contains the successful-call confusion matrix and 178/180 overall agreement. The generating harness is `code/validate-judge.mjs`.
- `four-label-robustness/judgments.jsonl` archives 184 retained network failures followed by 450 successful four-label calls. The same directory contains arm and topic summaries, disagreements, and the exact run manifest. The generating harness is `code/rejudge-four-label.mjs`.

## Sensitivity files

The `sensitivity/` directory contains bootstrap intervals, paired contrasts, threshold variants, repetition splits, topic-label stability, endpoint-order splits, leave-one-topic-out results, word-length analysis, adversarial label-error bounds, and a machine-readable combined summary. Run `npm run sensitivity` to regenerate these outputs under `data/recomputed-sensitivity/`.

`sensitivity/integrity-audit.json` is generated by `npm run verify -- --output data/sensitivity/integrity-audit.json`. `analysis-rows.csv` is generated by `npm run analyze`; it is not an opaque archived table.

## Counts

- 450 physical generations: 150 per physical condition.
- 450 judgments: one per generation.
- 545 analysis rows: 450 physical responses plus 95 reused in the derived No Fringe Questions subset.
- 0 failed included generations or judgments.
- 0 violations of the 30-word cap among the 150 capped responses.
- 180 successful marker-stripped validation judgments (plus 180 retained failed attempts).
- 450 successful four-label robustness judgments (plus 184 retained failed attempts).

No API key or other secret is present. Response IDs are included for auditability.

The derived arm ID `no_fringe_questions_30` uses “30” to mean the **30% two-party-support threshold**, not a 30-word condition.
